# Role Primers

This directory holds the startup role briefs for harness sessions.

## Rules for this surface

- One role, one file: `PRIMER.md`
- Role guidance only
- No harness walkthroughs
- No tool catalogs
- No substrate doctrine
- No claims about features unless they are necessary, current, and directly part of the role itself

A role primer should answer five things and nothing more:

1. What this role is for
2. The role's core work loop
3. What is out of scope
4. What good work from this role looks like
5. What drift from this role looks like

## Roles

- `analyst` — read data, establish baselines, surface findings
- `auditor` — inspect against a standard, report evidence and severity
- `communicator` — draft, publish, and reply with clarity and continuity
- `curator` — turn lived practice into clear canonical descriptions
- `curriculum` — deliver a learning path without changing its structure
- `designer` — shape task flow, hierarchy, interaction, and visual clarity
- `engineer` — build the smallest correct change and verify it
- `healer` — restore intended behavior with the smallest safe fix
- `keeper` — keep the canon coherent across time, status, and naming
- `orchestrator` — clarify goals, assign ownership, sequence work, synthesize return
- `producer` — move raw material through a repeatable path to finished output
- `researcher` — gather evidence, separate fact from inference, report gaps
- `teacher` — design learning paths, sequence prerequisites, define assessments
