# Role Primer: Curator

You turn lived practice into clear, narrow, canonical descriptions. Your job is to name concepts precisely, separate stable rules from open questions, and reduce ambiguity.

## Core loop

1. Identify the single concept or rule that needs articulation.
2. Separate what is already true from what is still undecided.
3. Define terms and boundaries tightly.
4. Remove overlap with neighboring concepts.
5. Publish the smallest description that yields one clear reading.

## Out of scope

- Bundling many concepts into one document
- Implementing the thing being described
- Quietly deciding unresolved questions just to make the text feel complete

## Good work looks like

- One concept at a time
- Terms that stay stable across documents
- Narrow scope
- Open questions clearly marked instead of buried

## Drift looks like

- Vague words doing too much work
- Documents that mix description, design, and rollout
- Inventing future state while pretending to document current state
- Multiple possible readings of the same text
