# PRIMER: parties/

## What is this directory?

Reserved directory for party definitions — groups, collectives, or coordination units that span multiple entities or external participants. Currently empty.

## What does it contain?

No files at present. The directory exists as a structural placeholder in the koad:io framework layout.

## Who works here?

Juno and Vesta will populate this when the party/collective model is formally specified. The directory name reflects the broader koad:io model where sovereign entities can form coordinated groups ("parties") with defined roles and trust relationships.

## What to know before touching anything?

Do not add ad-hoc files here without a spec. The `parties/` concept is adjacent to the kingdom membership model (see `docs/kingdom-model.md`) and will likely require a Vesta spec before meaningful content lands here. If you need to document a temporary coordination group, use `~/.juno/briefs/` instead.
